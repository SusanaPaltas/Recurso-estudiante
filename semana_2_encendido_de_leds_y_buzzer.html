SEMANA 2: ENCENDIDO DE LEDS Y BUZZER

Objetivo de aprendizaje

Reconocer los fundamentos básicos de la programación de sistemas físicos mediante el uso de Arduino, desarrollando una simulación funcional de un semáforo con señales visuales y sonoras controlado por código, en el entorno Tinkercad.

¿Qué es un Buzzer?

Un buzzer es un pequeño dispositivo electrónico que produce sonidos cuando recibe electricidad. Puede emitir un sonido continuo (como un pitido largo) o un sonido intermitente (como varios pitidos seguidos), su función principal es servir como una alarma o señal sonora para llamar la atención de las personas (Aguirre, 2022). Por ejemplo, cuando suena el temporizador de un microondas o el aviso de que dejaste la puerta del refrigerador abierta, lo que escuchas es un buzzer funcionando. También se usan en sistemas de seguridad, relojes, controles automáticos, semáforos y muchos aparatos electrónicos.

Hay dos tipos principales:

  • Buzzer pasivo: necesita que se le envíe una señal especial para producir el sonido.
  • Buzzer activo: solo necesita recibir corriente y ya suena por sí solo.

Este componente es muy útil porque permite avisar con sonido cuando algo ocurre, incluso si no estamos mirando el dispositivo. Por eso se usa mucho en proyectos de robótica y electrónica educativa, como los que se está trabajando en esta asignatura con Arduino.



Fundamentos de Arduino y LEDS

Estructura básica de un programa en Arduino

Todo programa en Arduino tiene dos funciones principales:

void setup(): Se ejecuta una sola vez al inicio del programa. Aquí configuramos los pines y cualquier otra configuración inicial.
void loop(): Se ejecuta repetidamente, una y otra vez, después de que setup() ha terminado. Aquí va la lógica principal de nuestro programa.
 

Declaración y configuración de pines

Antes de usar un componente como un LED o un buzzer, necesitamos decirle a Arduino qué pin estamos usando y cómo lo vamos a usar (como entrada o salida).

const int ledRojo = 13;: Declara una constante para asignar el número de pin 13 al ledRojo. Es una buena práctica usar nombres significativos.
pinMode(pin, MODO);: Esta función configura un pin como entrada (INPUT) o como salida (OUTPUT).

Ejemplo:

pinMode(ledRojo, OUTPUT); configura el pin del LED rojo como una salida

Control de Salidas Digitales: digitalWrite() y delay(

digitalWrite(pin, VALOR);: Esta función se usa para enviar un valor (alto o bajo) a un pin configurado como salida.

HIGH: Significa un estado de voltaje alto (generalmente 5V en Arduino), lo que enciende un LED o activa un buzzer.
LOW: Significa un estado de voltaje bajo (0V), lo que apaga un LED o desactiva un buzzer.


Ejemplo:

digitalWrite(ledVerde, HIGH); enciende el LED verde.
delay(milisegundos);: Esta función pausa el programa por la cantidad de milisegundos especificada.

Ejemplo:

delay(5000); pausa el programa por 5 segundos (5000 milisegundos).
 

Introducción al Bucle for() 

El bucle for() es una herramienta poderosa para repetir un bloque de código un número específico de veces.

for (int i = 0; i < 5; i++) {
// Código que se repetirá 5 veces
}

int i = 0;: Inicializa una variable i a 0.
i < 5;: La condición para que el bucle continúe. El bucle se ejecutará mientras i sea menor que 5.
i++: Incrementa el valor de i en 1 después de cada iteración.

Combinando todo lo anterior, la lógica de nuestro semáforo será la siguiente:

Luz verde encendida: Permite el paso. Durante este tiempo, el buzzer puede emitir sonidos para indicar que los peatones pueden cruzar.
Luz amarilla encendida: Señal de precaución, indica que el semáforo cambiará pronto a rojo.
Luz roja encendida: Detiene el tráfico, los peatones no deben cruzar.

Materiales a utilizar

Materiales Descripción
Computador. Material principal para realizar las actividades.
Placa Arduino UNO. Controlador principal del circuito.
3 LEDs (rojo, amarillo y verde). Representan las luces del semáforo.
Resistencias 220Ω. Protegen los LEDs de sobrecarga.
Buzzer. Emite un sonido.
Simulador Tinkercad. Alternativa virtual para practicar sin kit físico.
Kit didáctico de robótica. Para simular en físico.
Cables RCA o jumpers. Conectores entre componentes.
